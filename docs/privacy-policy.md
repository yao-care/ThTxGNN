# นโยบายความเป็นส่วนตัว

**อัปเดตล่าสุด**: 27 กุมภาพันธ์ 2569

---

## ภาพรวม

<p class="key-answer" data-question="ThTxGNN จัดการกับข้อมูลของฉันอย่างไร?">
ThTxGNN SMART App เป็น<strong>แอปพลิเคชันฝั่งไคลเอนต์เท่านั้น</strong> การประมวลผลข้อมูลทั้งหมดทำในเบราว์เซอร์ของคุณ เรา<strong>ไม่เก็บรวบรวม ไม่จัดเก็บ ไม่ส่ง</strong>ข้อมูลสุขภาพส่วนบุคคล (PHI) ใดๆ ไปยังเซิร์ฟเวอร์ของเรา
</p>

---

## การเก็บรวบรวมและการใช้ข้อมูล

### ข้อมูลที่เราไม่เก็บรวบรวม

| ประเภทข้อมูล | สถานะการเก็บรวบรวม |
|-------------|-------------------|
| ชื่อผู้ป่วย วันเกิด ข้อมูลติดต่อ | **ไม่เก็บรวบรวม** |
| ประวัติการใช้ยาของผู้ป่วย | **ไม่เก็บรวบรวม** |
| บันทึกการวินิจฉัย | **ไม่เก็บรวบรวม** |
| ข้อมูลสุขภาพส่วนบุคคล (PHI) ใดๆ | **ไม่เก็บรวบรวม** |

### แอปพลิเคชันทำงานอย่างไร

1. **การอนุญาต EHR**: คุณเปิดใช้งาน ThTxGNN SMART App จากระบบเวชระเบียนอิเล็กทรอนิกส์ (EHR)
2. **การประมวลผลในเครื่อง**: แอปพลิเคชันอ่านข้อมูลยาในเบราว์เซอร์ของคุณ
3. **การค้นหาในเครื่อง**: การแมปยาและการค้นหาข้อบ่งใช้ใหม่ทำในเบราว์เซอร์ทั้งหมด
4. **ไม่มีการส่งข้อมูลออก**: ข้อมูลผู้ป่วยไม่ออกจากเบราว์เซอร์ของคุณ

## สถาปัตยกรรมทางเทคนิคและมาตรการรักษาความปลอดภัย

### สถาปัตยกรรมฝั่งไคลเอนต์เท่านั้น

<p class="key-answer" data-question="สถาปัตยกรรมทางเทคนิคของ ThTxGNN คืออะไร?">
ThTxGNN SMART App ใช้สถาปัตยกรรม<strong>ฝั่งไคลเอนต์เท่านั้น (Client-Side Only)</strong>: โค้ดทั้งหมดทำงานในเบราว์เซอร์ของผู้ใช้ ไม่ต้องการเซิร์ฟเวอร์ประมวลผลข้อมูลผู้ป่วย
</p>

| รายการ | คำอธิบาย |
|--------|----------|
| การประมวลผลฝั่งเซิร์ฟเวอร์ | ไม่มี |
| ฐานข้อมูล | ไม่มี (ใช้ไฟล์ JSON แบบคงที่) |
| การจัดเก็บข้อมูลผู้ป่วย | ไม่มี |
| การจัดเก็บ Session | เฉพาะในหน่วยความจำเบราว์เซอร์ ลบเมื่อปิด |

### การอนุญาตด้วย OAuth 2.0

แอปพลิเคชันนี้ใช้ **OAuth 2.0 with PKCE** (Proof Key for Code Exchange) สำหรับการอนุญาต:

- **การป้องกัน PKCE**: ป้องกันการโจมตีการดักจับรหัสอนุญาต
- **หลักการสิทธิ์ขั้นต่ำ**: ขอเฉพาะสิทธิ์การอ่านที่จำเป็น
- **ไม่มีข้อมูลรับรองระยะยาว**: ไม่จัดเก็บ refresh token
